Подробнее:

Делая проект, я пытался сделать фокус на архитектуре приложения (это и съедало мое время в основном). Не исключено, что с точки зрения архитектуры тут есть большие изъяны. Помимо обычно реализации компонентов материала, на проекте реализован динамический заголовок, завизимый от текущей страницы. Директивы можно увидеть в попапе, который вызывается в момент ошибки http запроса и в песочнице. Интерсептор перехватывает все http запросы и переносит их в стор приложения. У стора есть абстрактный класс, согласно которому тот должен реализовывать свое поведение. Проект также использует ленивую загрузку модулей с настроеным прелоудом
